| `roomUrl` | RadGridExport | `processRoomRows()` |
| `teachersUrl` | Teachers sheet | `processTeacherRows()` |

- Branch data derived from Room Schedule events, overridden by Teachers sheet entries
- Instrument tags derived at render time from `state.lf.records` (no extra column needed)
- Room/Teachers fetch failures caught gracefully so Lesson Finder still works if they fail

## 5. Bug fix — Room Schedule URL not persisting to localStorage

The save block in the connect-btn handler used `if (roomUrl)` guards, which skipped saving when the field was empty. Fixed to unconditionally save/remove all three keys:

```js
localStorage.setItem(LS_LESSON_URL, lessonUrl);
roomUrl     ? localStorage.setItem(LS_ROOM_URL,     roomUrl)     : localStorage.removeItem(LS_ROOM_URL);
teachersUrl ? localStorage.setItem(LS_TEACHERS_URL, teachersUrl) : localStorage.removeItem(LS_TEACHERS_URL);
```

## 7. Cache-busting for Google Sheets fetches

Added timestamp parameter and `cache: 'no-store'` to `fetchCsvUrl()`:

```js
async function fetchCsvUrl(url) {
  const bust = (url.includes('?') ? '&' : '?') + '_=' + Date.now();
  const resp = await fetch(url + bust, { cache: 'no-store' });
  ...
}
```

Bypasses the browser HTTP cache on every refresh. Note: Google's CDN-level cache (separate from the browser cache) still propagates updates with a ~5–10 minute delay regardless.
